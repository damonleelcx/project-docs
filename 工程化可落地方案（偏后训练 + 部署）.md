# 一、核心思路（你真正要做的事）

意图识别 ≠ 一定要用大模型

本质是一个分类问题：

```
输入: "我想退款"

```

- 低延迟（<10ms）
- 高QPS
- 可持续训练

# 二、方案选型（推荐优先级）

## ✅ 方案1：Embedding + 相似度（最简单 & 很快）

### 思路

1. 用一个小 embedding 模型（本地）
2. 每个 intent 准备几个样本
3. 做向量相似度匹配

```
用户输入 → embedding → 向量
               ↓
       和 intent 样本做 cosine similarity
               ↓
           取最大
```

### 优点

- 无需训练（或只需少量调优）
- 推理极快
- 易扩展

### 模型推荐（小模型）

- bge-small-en
- e5-small
- 中文可用：
  - bge-small-zh

---

### Go 怎么做？

#### 架构

```
[Golang API]
     ↓
[Embedding Service (Python / Rust / ONNX)]
     ↓
[Vector Search (in-memory / FAISS)]
```

#### Go 调用方式

- HTTP 调 embedding 服务
- 或用 ONNX Runtime 直接本地跑

👉 Go库：

- onnxruntime-go

# 三、方案2：小分类模型（推荐生产）

如果你要更准一点👇

## 模型选择

- DistilBERT
- MiniLM

---

## 训练流程（核心）

### 1️⃣ 数据准备

```json
{"text": "我要退款", "label": "refund"}
{"text": "帮我查订单", "label": "query_order"}
{"text": "取消订单", "label": "cancel"}
```

---

### 2️⃣ 微调（Python）

用 HuggingFace：

```python
from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained(
    "distilbert-base-uncased",
    num_labels=3
)
```

---

### 3️⃣ 导出 ONNX

```bash
python -m transformers.onnx --model=./model onnx/
```

---

### 4️⃣ Go 推理

```go
import "github.com/microsoft/onnxruntime-go"

session.Run(...)
```

---

## 性能

- 延迟：5~20ms（CPU）
- QPS：几百 ~ 上千（单机）

👉 完全可以做在线服务

# 四、方案3：规则 + 模型（工业最优解）

真实生产里通常这样：

```
if 命中规则:
    return intent
else:
    用模型预测
```

---

### 规则例子

```go
if strings.Contains(text, "退款") {
    return "refund"
}
```

---

### 为什么要这样？

- 80% 高频请求 → 规则（0ms）
- 长尾 → 模型

👉 延迟直接下降一个数量级

---

# 五、训练优化（后训练重点）

你说“模型后训练”，重点在这里👇

## 1️⃣ Hard Negative Mining

```
"我要退款" vs "我要取消订单"
```

👉 很像但不同 → 提高区分能力

---

## 2️⃣ 数据增强

- 同义词替换
- paraphrase（可用 LLM 离线生成）

---

## 3️⃣ 蒸馏（强烈推荐）

用大模型标注数据：

```
GPT-4 → 打标签
↓
训练小模型
```

👉 小模型效果接近大模型
